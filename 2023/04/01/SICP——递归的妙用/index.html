Structure and Interpretation of Computer Programs

Good morning! Today, I want to talk something about recursion. Before that, I would say that computer sciencs is a branch of mathematics.

So what is recursion?

Okay, if you want to understand recursion, you should first know about what is recursion. The progress is just like peeling an onion and restoring it. (However, I think this progress what we learn is hard to solve it.)

I like a Chinese story and I think it can be used to interpret what is recursion: 从前有座山，山中有座庙，庙里有个老和尚，老和尚在给小和尚讲故事：“ 从前有座山，山中有座庙，庙里有个老和尚，老和尚在给小和尚讲故事：“ 从前有座山，山中有座庙，庙里有个老和尚，老和尚在给小和尚讲故事：“ 从前有座山，山中有座庙，庙里有个老和尚，老和尚在给小和尚讲故事：“太困了不讲了”，于是都回去睡觉了。”于是都回去睡觉了。”于是都回去睡觉了。”于是都回去睡觉了。

So why some engineers like to ues recursion instead of iteration? What's more, recursion can result in stack overflows and extremely unstable time complexity(This can sometimes be a disaster).

One bad apple won't spoil the whole bunch. Recursive program features include elegance, inverse and recursive relationships

Elegence: If you use recursion well, you can reduce what shit-mountain you write. It can make people to know what you write easily.

Inverse: Since the recursive caller needs to maintain the call stack, and the stack has the characteristics of last-in, first-out, the recursive program is suitable for meeting the requirements of the negation class.

Recursive relationship: This can sometimes be a good interpretation of mathematical induction (finding the original term and listing the recursive).

How do we use recursion to solve some interesting problem? Let's see some examples.

Question 1: the sum of any one-digit non-negative number is that number.

Question 2: The factorial of a number is defined as \(n! .\)

Summery

If we want to solve recursive problem, we should find Base case , Recursive Case and Conditional statement. > Base case: Evaluated without a recursive call (the smallest subproblem).

Recursive case: Evaluated with a recursive call (breaking down the problem further)

Conditional statement to decide if it's a base case
